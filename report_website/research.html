Research

Project Research

Stakeholder Insights

This project was developed in collaboration with Avanade and supported by insights from the NHS. From early-stage stakeholder interviews, we identified several key expectations:

  • Long-term memory retention via Retrieval-Augmented Generation (RAG) (Lewis et al., 2020)
  • A simplified, avatar-based interface suitable for elderly users
  • Voice input and output for accessibility
  • A privacy-respecting offline mode for local use
  • Avoiding hallucinations in healthcare-related conversations

These requirements helped shape our system architecture and informed both technical and design decisions throughout development.

Recent studies suggest that Al-based conversational agents can play a significant role in supporting mental health and reducing loneliness among users. Fitzpatrick et al. (2017)demonstrate that automated conversational agents, such as Woebot, can effectively deliver cognitive behavioral therapy, significantly reducing symptoms of depression and anxiety.Inspired by these findings, Companio incorporates emotional empathy and cognitive assistance features, aiming to provide similar psychological benefits to elderly users and those experiencing loneliness.

Existing Solution Evaluation

During development, we iterated on multiple approaches and tested several tools:

  • F5-TTS offered realistic voice synthesis but was too heavy for offline deployment.
  • React was selected for its flexibility and performance with dynamic interfaces.
  • Qdrant was compared against traditional databases and chosen for its RAG performance locally.
  • We refactored our initial backend into modular services, applying proven software patterns like Strategy, Adapter, and Singleton to maintain code clarity.

Through these trials, we ensured the system could scale while staying maintainable and accessible across various user environments.

Design Decisions Informed by Research

  • Simple navigation and large visual elements for accessibility
  • Clear voice/text feedback with minimal cognitive load
  • Avatar interaction for a personal, warm experience

We applied Nielsen?s Ten Usability Heuristics in UI design (Nielsen, J. ,1994), conducted a heuristic evaluation, and evolved our wireframes accordingly. Real-world testing (e.g., school demos, internal showcases) confirmed our choices, especially the addition of memory games, chat history, and multi-modal interaction.

Technology Research & Selection

To support both performance and accessibility, we evaluated a range of technologies before finalizing our stack:

Component Technology Used Reason for Choice
AI Model (Online) Azure OpenAI (GPT)(Brown et al., 2020) High quality, reliable, and easy to integrate
AI Model (Offline) IBM Granite / Microsoft Phi-3.5 (IBM Research, 2025) Lightweight, efficient, and suitable for local inference
Vector DB (Online) Azure AI Search (Microsoft Azure, 2025, Cormack et al. 2009) Supports hybrid search (vector + keyword), scalable and cloud-based
Vector DB (Offline) Qdrant (Qdrant, 2025) + FastEmbed (Docker Inc., 2025) Enables hybrid RAG locally, Docker-hosted and open-source
Embedding Services AzureTextEmbedding (OpenAI, 2025), GraniteEmbeddingService Seamless integration with Semantic Kernel
Orchestration Semantic Kernel (Microsoft) (Microsoft ? 2025) Modular, extensible AI integration framework
Frontend Framework React + Three.js (Cabello, 2025) Dynamic, customizable avatar rendering
TTS/STT (Online) Azure Speech Services (Microsoft Azure, 2025) High-quality cloud-based voice interaction
TTS/STT (Offline) Coqui TTS (Coqui AI, 2025), browser STT (MDN, 2025) Lightweight, open-source, suitable for local usage
