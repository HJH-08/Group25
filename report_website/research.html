Research

Stakeholder Insights

This project was developed in collaboration with Avanade and supported by insights from the NHS. From early-stage stakeholder interviews, we identified several key expectations:

  • Long-term memory retention via Retrieval-Augmented Generation (RAG) (Lewis et al., 2020)
  • A simplified, avatar-based interface suitable for elderly users (Nielsen, 1994)
  • Voice input and output for accessibility
  • A privacy-respecting offline mode for local use
  • Avoiding hallucinations in healthcare-related conversations

These requirements helped shape our system architecture and informed both technical and design decisions throughout development.

Recent studies suggest that AI-based conversational agents can play a significant role in supporting mental health and reducing loneliness among users. Fitzpatrick et al. (2017) demonstrate that automated conversational agents, such as Woebot, can effectively deliver cognitive behavioral therapy, significantly reducing symptoms of depression and anxiety. Inspired by these findings, Companio incorporates emotional empathy and cognitive assistance features, aiming to provide similar psychological benefits to elderly users and those experiencing loneliness.

RAG Algorithm

During our research, we explored various algorithms to optimise our RAG implementation for both performance and efficiency. Initially, we used direct vector search?matching user input against a vector database of past conversations. While this worked well for general context retrieval, it struggled with precise text lookups like names or dates.

To address this, we adopted a hybrid search approach that combined vector similarity with BM25 keyword-based search[20]. This method was deployed across both our online and offline services. Azure AI Search supported both techniques out of the box, making integration straightforward. However, implementing this in Qdrant required workarounds due to limitations in Semantic Kernel's native support[21].

In our offline setup, we built a multi-vector system in Qdrant using:

  • Standard embeddings from IBM Granite
  • Sparse BM25-style embeddings via Qdrant
  • Late-interaction embeddings for reranking

This hybrid approach significantly enhanced retrieval accuracy and relevance, resulting in more contextually rich and precise chatbot responses.

We also investigated a more advanced framework: Retrieval-Augmented Fine-Tuning (RAFT), which integrates RAG with model training[22]. While early research showed strong potential, we couldn't proceed due to the challenge of sourcing and cleaning a large, consistent chat history dataset. For this reason, we continued with our hybrid retrieval solution.

Memory Games

Our choice of memory mini games are backed by extensive research into the memory-enhancing performance of each game. For Match the Pairs, a study involving primary students showed that participants playing this memory game demonstrated notable improvements in academic performance compared to a control group. The study concluded that the memory games enhance critical cognitive skills, including attention, concentration, and visual recognition, thereby positively influencing overall academic achievement[23].

Another article demonstrated how a game based on Simon can facilitate mental rehearsal of newly acquired information. The study observed two participants engaging in a sequence-replication task similar to the Simon Game and found that their brains continued to replay the learned sequences during rest periods. This act of replay is believed to contribute to the consolidation of new memories, suggesting that engaging in such games can enhance memory retention and retrieval processes[24].

Existing Solution Evaluation

During development, we iterated on multiple approaches and tested several tools:

  • F5-TTS offered realistic voice synthesis but was too heavy for offline deployment.
  • React was selected for its flexibility and performance with dynamic interfaces.
  • Qdrant was compared against traditional databases and chosen for its RAG performance locally.
  • We refactored our initial backend into modular services, applying proven software patterns like Strategy, Adapter, and Singleton to maintain code clarity.

Through these trials, we ensured the system could scale while staying maintainable and accessible across various user environments.

Design Decisions Informed by Research

User feedback and stakeholder interviews revealed key UI needs:

  • Simple navigation and large visual elements for accessibility
  • Clear voice/text feedback with minimal cognitive load
  • Avatar interaction for a personal, warm experience

We applied Nielsen's Ten Usability Heuristics in UI design (Nielsen, J., 1994), conducted a heuristic evaluation, and evolved our wireframes accordingly. Real-world testing (e.g., school demos, internal showcases) confirmed our choices, especially the addition of memory games, chat history, and multi-modal interaction.
